## 1.关于github pages

* github pages是github为了更好介绍项目和个人而开放的，支持个人和项目静态网页展示，因此被很多人用来做个人博客使用；随之的也有很多组件供我们使用，推荐一个**HEXO**，之前在折腾github pages的时候也曾经使用过，是书写和生成博客静态页面的利器；有兴趣的小伙伴可以去尝试一下；后面我会把之前介绍hexo的博文也上传一份；

* [github pages的官网介绍首页](https://pages.github.com)
* [hexo 首页](https://hexo.io/)

## 2.迁移步骤

* 将github pages个人主页仓库克隆到本地 
* 建立一个文件夹blogs专门用来存放leanote导出的静态html文件和相应的资源
* 新建一个文件index.json专门用来记录博客历史（由于github只支持静态页面，博客记录自己维护，记录博客名，博客时间，博客标签）
    ```
        {
          "data": [
        	["groovy初体验","2017/08/08",["groovy"]],
        	["数据库迁移小问题汇总","2017/08/01",["数据库迁移","mysql","tdh"]],
        	["DruidDataSource","2017/08/05",["druidDataSource","数据源"]],
        	["旧外部数据采集系统说明","2017/08/02",["celery","外部数据采集"]],
        	["新外部数据采集系统架构说明","2017/08/03",["celery","外部数据采集"]],
        	["index","2016/08/08",["blog","index"]]
          ]
        }
    ```
* 首页index.html文件中采用[dataTables.js](http://www.datatables.club/)插件动态读取上面的博客历史记录文件，维护对应的链接地址即可；

## 3.博客迁移步骤
* leanote导出博客为html文件和对应的资源目录
* 修改html文件中的链接地址；主要是因为http和https不能混用，外链都修改为https
* 将博客html和资源文件目录同时放在blogs文件夹中
* 修改index.json文件，新增一条对应的记录即可；注意标题和html文件名不能错
