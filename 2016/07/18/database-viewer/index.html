数据库视图小记

Jul 18, 2016 | database | Hits

本文对数据库的视图的定义和使用做一些总结。

最近手头上的电信诈骗项目有这样一个需求：增加一个单纯的查询页面，但是查询内容包含了好几笔之前写好的交易的内容，因此需要到不同的数据表中查询数据在前台进行展示，而且并不需要展示全部的字段，综合考虑后，决定采用数据图视图来进行查询。

视图的定义

在 SQL 中，视图是基于 SQL 语句的结果集的可视化的表。

视图包含行和列，就像一个真实的表。视图中的字段就是来自一个或多个数据库中的真实的表中的字段。我们可以向视图添加 SQL 函数、WHERE 以及 JOIN 语句，我们也可以提交数据，就像这些来自于某个单一的表。

(也就是说其实我们可以直接将视图当做一个普通的数据表，这个表中的字段可以来自于多个表或者sql函数，同时视图是一种逻辑对象，可以理解为数据表的影子，他不占用物理存储空间）

视图的作用

  • 视图的定义既然就是一个数据表，作为一个普通的表来使用也就无可厚非了；
  • 对于需要经常查询的条件语句可以预定义为视图，存储在数据库中，通过访问视图来查询满足这条查询语句的数据；
  • 视图的内容可以基于某个表的列子集或者行子集=====可以利用这个特性用来隐藏主表中部分字段和数据；
  • 视图的内容可以基于多个表的联合或链接=====可以利用这个特性来方便联合查询多个表的数据，能够轻松组织满足应用的数据结构；
  • 视图的内容同时还可以基于视图或者视图和表======这提供了很大的灵活性，让应用的数据库的设计变得更加方便和灵活；
